HTML Essentials

1. HTML Basics & Structure

HTML provides the structure for web pages using elements and tags.

HTML Boilerplate

The basic structure every HTML document should have:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
</head>
<body>
    
</body>
</html>

Headings, Paragraphs, and Lists

HTML offers six levels of headings (<h1> to <h6>), paragraph tags (<p>), and three types of lists: ordered (<ol>), unordered (<ul>), and description lists (<dl>).

Links, Images, and Attributes

<a> tags create hyperlinks, while <img> tags embed images. Attributes provide additional information about elements.

Tables

Tables organize data into rows and columns using <table>, <tr>, <td>, and <th> tags. colspan and rowspan attributes merge cells.

Multimedia

<audio> and <video> tags embed media, while <iframe> embeds external content.

2. Forms + Accessibility Essentials

Forms collect user input, while accessibility ensures everyone can use your website.

Input Types

HTML offers various input types: text, email, password, checkbox, radio, date, file, and more.

Labels and Grouping

<label> tags improve form accessibility. <fieldset> and <legend> group related form elements.

Buttons

Buttons perform actions: submit forms, reset them, or trigger custom functionality.

Accessibility

alt text describes images for screen readers. ARIA attributes enhance accessibility. Keyboard navigation ensures usability without a mouse.

3. Semantic HTML

Semantic HTML uses meaningful tags that describe their purpose.

Semantic Elements

<header>, <nav>, <main>, <article>, <section>, <aside>, and <footer> provide structure and meaning.

SEO and Accessibility Benefits

Semantic HTML improves search engine rankings and makes content more accessible to screen readers.

CSS Fundamentals

1. CSS Basics & Selectors

CSS styles HTML elements using various selector types and follows the box model.

CSS Implementation

CSS can be added inline, internally with <style> tags, or externally via <link> to a .css file.

Selectors

Selectors target elements: universal (*), type, class (.), id (#), attribute, and pseudo-classes (:hover, :focus).

Box Model

Every element is a box with content, padding, border, and margin areas.

Units

CSS supports various units: px (pixels), em, rem, % (percentage), vh (viewport height), vw (viewport width).

2. Layouts

CSS provides multiple layout techniques for organizing content on the page.

Display Property

The display property controls layout: block, inline, inline-block, and none.

Positioning

Elements can be positioned using: static, relative, absolute, fixed, and sticky.

Flexbox

Flexbox provides efficient way to lay out, align, and distribute space among items in a container.

3. CSS Polish

Advanced CSS features create engaging, responsive user experiences.

Transitions

Transitions smoothly animate property changes over specified durations.

Transforms

Transforms rotate, scale, translate, or skew elements in 2D or 3D space.

Animations

@keyframes create complex animations with multiple steps and property changes.

Responsive Images

max-width: 100% ensures images scale properly. <picture> and srcset provide art direction.

Media Queries

@media rules apply different styles based on device characteristics like screen width.

J

JavaScript Core

1. Basics

JavaScript adds interactivity and dynamic behavior to web pages.

Variables

Variables store data: var (function-scoped), let and const (block-scoped).

Data Types

JavaScript has primitive types: string, number, boolean, null, undefined, and complex types: object, array.

Operators

Operators perform operations: arithmetic, comparison, logical, and assignment.

2. Functions, Scope & Objects

Functions encapsulate logic, while objects store related data and functionality.

Functions

Functions can be declared, expressed, or written as arrow functions. They accept parameters and return values.

Scope

Scope determines variable accessibility: global, local, block. Closures preserve outer function variables.

Objects

Objects store key-value pairs with properties and methods. "this" refers to the object context.

Object Manipulation

Properties can be accessed with dot or bracket notation. JSON formats object data for transmission.

3. DOM and Events

The Document Object Model represents the page structure that JavaScript can manipulate.

DOM Structure

The DOM is a tree of nodes: document, elements, text nodes, etc.

Selecting Elements

Elements can be selected by ID, class, tag name, or CSS selectors.

Changing Content

innerHTML, textContent, setAttribute, and style properties modify elements.

Creating/Removing Elements

createElement, appendChild, and remove dynamically alter the DOM structure.

Event Handling

Events like clicks, inputs, and keypresses can be handled with event listeners.

Developer Tools & Productivity

1. GitHub Basics

GitHub is a platform for version control and collaboration using Git.

Repositories

Repositories store project files and their revision history.

Clone, Push, Pull

Clone copies repositories, push uploads changes, pull downloads updates.

Branching & Merging

Branches isolate development work. Merging combines branch changes.

Pull Requests & Code Reviews

Pull requests propose changes. Code reviews ensure quality before merging.

Merge Conflicts

Conflicts occur when Git can't automatically merge changes and require manual resolution.

2. VS Code Power Usage

VS Code is a popular code editor with powerful features for developers.

Emmet Shortcuts

Emmet abbreviations expand to full HTML/CSS code structures, speeding up development.

Snippets

Custom code templates can be created and reused with shortcut triggers.

Extensions

Extensions like Prettier, Live Server, and ESLint enhance functionality and workflow.

Debugging

Breakpoints pause execution. The console logs information and errors.

3. DevTools + Debugging

Browser DevTools help inspect, debug, and optimize web pages.

Inspect Element

Examine and modify HTML/CSS in real-time to see changes instantly.

Box Model Visualization

DevTools visually represent the box model, showing margin, border, padding, and content.

Debugging JavaScript

Set breakpoints, step through code, and inspect variables to find and fix issues.

Performance Tab

Analyze load times and runtime performance to identify bottlenecks.

Network Tab

Monitor API requests, examine headers, and check caching behavior.

Console Tricks

Beyond console.log(), use console.table(), console.error(), and other methods for better debugging.
